# 评分标准与权重

## 固定权重

| 维度 | 权重 | 否决触发条件 |
|-----------|--------|--------------|
| Policy | 22% | 分数 < 60 → 不通过 |
| E-E-A-T | 17% | — |
| Content | 15% | — |
| Cookie | 13% | — |
| Ad Placement | 10% | — |
| Traffic | 8% | — |
| Tech | 8% | — |
| Legal | 7% | — |

## 等级划分

| 总分 | 等级 | 风险级别 |
|-------------|-------|------------|
| 90–100 | A | 低 |
| 80–89 | B | 低 |
| 70–79 | C | 中 |
| 60–69 | D | 中 |
| 0–59 | F | 高 |

## 分数归一化

每位专家报告原始分数 0 到 100。协调器计算：

```
weighted = rawScore * weight
totalScore = sum(all weighted scores)
```

所有分数在 `report-final.json` 中保留一位小数，在终端输出中四舍五入为整数。

## 各维度状态阈值

| 分数 | 状态 | 含义 |
|-------|--------|---------|
| 90–100 | 通过 | 优秀，无阻塞性问题 |
| 70–89 | 警告 | 可接受，但需小幅改进 |
| 60–69 | 警告 | 有风险，强烈建议改进 |
| 0–59 | 不通过 | 存在阻塞性问题，AdSense 获批可能性低 |

## 严重级别

每位专家报告中的发现项使用以下严重级别：

- **critical（严重）** — 直接违反 AdSense 政策或可能导致被拒的原因。申请前必须修复。
- **warning（警告）** — 削弱申请竞争力的风险因素。1 周内修复。
- **info（信息）** — 建议或最佳实践缺口。方便时修复。

## 否决规则

`ads-policy-expert` 拥有否决权。如果其分数低于 60，整个审计将收到：
- 等级：**F**
- 风险：**高**
- 建议："在解决政策违规问题之前，请勿申请 AdSense。"
